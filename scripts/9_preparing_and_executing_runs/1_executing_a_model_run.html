Executing a model run

This guide explains how to execute the command-line version of NewDepomod

Running the model

Executing a model simulation is straightforward once a model run object has been instantiated (and configured as required).
projectDir = 'C:\newdepomod_projects\bay_of_fish';
project = Depomod.Project.create(projectDir);
run = project.solidsRuns.first;
run.execute;
Starting simulation...
Command: "C:\Program Files\depomodruntimecontainer\bin\depomodruntimecontainer" --modelRunTimeFile "C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-1-NONE.depomodruntimeproperties" --nosplash --singleRunOnly --showConsoleOutput --showConsoleProgress
Time: 10-Jun-2019 09:05:19
Elapsed time is 1.038714 seconds.
By default, this will open a command prompt window within which the model will be executed. This means that NewDepomod does not tie up Matlab and separate work within Matlab can continue. Notice that the elapsed time it outputted. This simply reflects the time taken to hand the execution of the model to a process outside of Matlab.

Running within Matlab

The model can be executed directly within the Matlab environment. This has the advantage that Matlab knows when the run has completed and can then take subsequent action, such as processing output files or initiating another run of the software.
To run the model within the Matlab environment we make a simple change.
run.execute('runInBackground', 0);
Starting simulation...
Command: "C:\Program Files\depomodruntimecontainer\bin\depomodruntimecontainer" --modelRunTimeFile "C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-1-NONE.depomodruntimeproperties" --nosplash --singleRunOnly --showConsoleOutput --showConsoleProgress
Time: 10-Jun-2019 09:29:23
Elapsed time is 24.094584 seconds.
In this case, no command prompt window is opened and Matlab continues to be tied up by the execution of the model. Notice the elapsed time in this case is longer and includes the duration of the model run (the duration simulated was just a single day - most runs take much longer than this).
