Synchronising inputs and cages

This guide explains how Matlab can be used to ensure that inputs files are consistent with cage files.

About

Each cage in a cage file has a checksum style identifier stored in the inputsId property, e.g.
run.cages.cageGroups{1}.cages{1}
ans =
Circle with properties:

x: 350413
y: 1068926
length: 31.84
width: 31.84
height: 12
depth: 6
inputsId: '4c97b36c-bd67-44db-bfe7-15d6b3b298d2'
proportion: 0.125
inProduction: 1
This value should be the same for all cages in the cage definition file.
Each inputs file has a property that describes the associated cage inputs id
run.inputsFile.FeedInputs.uuid
ans = '4c97b36c-bd67-44db-bfe7-15d6b3b298d2'
It is essential that these values correspond otherwise the software will not run a successful simulation. There is a convenient method in the Matlab library for ensuring this.
run.syncCageID
It is good practice to invoke this method when setting up a model run, particularly in cases wherre new cages have been defined. There is no need to explicitly save the inputs file when invoking this method as this is done within the method itself.
