Calculating the total discharge

This guide explains how to calculate the total quantity discharged using the inputs properties file

About

Calculating the total quantity discharged can be useful for providing context to a modelling scenario. For example, it can be used to contextualise the amount of material remaining within the vicinity of the farm - i.e. as a proportion of the total discharged.
The NewDepomod software does provide summary statistics about material released and material remaining within the model domain at the end of the model. These values can be calculated manually using the respective model run inputs file (for the material released) and sur file(s) (for the material in the model domain). In some cases it can be preferable to calculate these manually - for example to resolve these questions throughout the model simulation rather than simply at the end.
This guide explains how to (very simply) calculated the tot discharged quantities.

Calculating total discharges

To calculate the total quantity discharged it is a simple matter of summing the columns in the inputs file discharge time series data table. Care needs to be taken in selecting the correct columns. The essentially work in pairs for any given discharge type: a column for waste feed and a column for faeces.
  • Columns 1 (waste feed) and 4 (faeces) for total organic solids
  • Columns 2 and 5 for carbon
  • Columns 3 and 6 for EmBZ
Calculating the total discharge is simply a matter of summing the two columns which are pertinent to the discharge we're interested in.
For example, to calculate the total organic solids discharge through a model run we can do this.
totalOrganicsDischarge = sum(inputs.data(:,1)) + sum(inputs.data(:,4))
totalOrganicsDischarge = 408046.274999982
In other words, we summing total waste feed and the total faeces to arrive at 408,046 kg.
We can check this against the value in the log file
run.log.Masses.solids.released.run
ans = '408046275.0000'
which is in g rather than kg, so we can see that the values match.

Calculating time-specific cumulative discharges

The value in calcuating the total discharge this way is that we can discover the corresponding values at any point in the model simulation. Currently, the model is configured (in the model properties file) to run for
run.runDurationDays
ans = 365
Just to check for consistency, we can check the implied run length in the inputs file
inputs.FeedInputs.numberOfTimeSteps
ans = '8760'
size(inputs.data,1)
ans = 8760
So the model is set for a 365 day run (according to the model properties file) and 8760 hours in the inputs file. These are equivalent.
Suppose we want to know the total cumulative discharge over the first 90 days, we can calculate this from the inputs file discharge time series data table. First we need to know how many rows to sum over, and since they each represent hours we just need to know how many hours are in 90 days.
90*24
ans = 2160
Okay, easy. Now we can do our cumulative discharge calculation
cumulativeOrganicsDischarge = sum(inputs.data(1:2160,1)) + sum(inputs.data(1:2160,4))
cumulativeOrganicsDischarge = 100614.149999998
This is about a quarter of the previous value which makes sense
