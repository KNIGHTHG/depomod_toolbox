Reading a model inputs file

This guide explains how to use Matlab to read a model run inputs properties file

About

The inputs file defines characteristics of the discharges that arise from the farm cages - i.e. waste feed and faeces.
The file is split into two sections: an upper "header" comprising a series of properties; and a subsequent CSV-style data table which describes time series discharge information.

Reading a properties file

Any NewDepomod properties file can be read in individually (see below) but the easiest method is to use the helper methods on a respective model run object.
For a given mode run, the inputs file can be found at:
run = project.solidsRuns.number(1);
run.inputsFilePath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\inputs\bay_of_fish-1-NONE-allCages.depomodinputsproperties'
The file can be instantiated in Matlab using the similar method
inputs = run.inputsFile
inputs =
InputsPropertiesFile with properties:

dataColumnCount: 6
run: [1×1 NewDepomod.Run.Solids]
data: [8760×6 double]
path: 'C:\newdepomod_projects\bay_of_fish\depomod\inputs\bay_of_fish-1-NONE-allCages.depomodinputsproperties'
endOfDataMarker: 'endOfDataMarker'
startOfDataMarker: 'startOfDataMarker'
Particle: [1×1 struct]
FeedInputs: [1×1 struct]
This returns an InputsPropertiesFile object which can be navigated and maniputated in a variety of ways.

Input file properties

The properties described in the file can be read individually. The Matlab syntax is as follows.
inputs.FeedInputs.biomass
ans = '1000'
inputs.FeedInputs.numberOfTimeSteps
ans = '8760'
inputs.FeedInputs.deltaT
ans = '3600.000000'
inputs.Particle.settlingVelocityOfFaeces
ans = struct with fields:
dispersion: '0.003500'
distribution: 'UNIFORM'
location: '-0.035000'
inputs.Particle.settlingVelocityOfFeed
ans = struct with fields:
dispersion: '0.009500'
distribution: 'UNIFORM'
location: '-0.095000'
inputs.Particle.settlingVelocityOfFaeces.location
ans = '-0.035000'
Note that all values are returned as char variables.

Data table

The table of discharge data can be returned using
inputs.data
ans = 8760×6
7.9625 3.901625 0 38.618125 11.5854375 0
7.9625 3.901625 0 38.618125 11.5854375 0
7.9625 3.901625 0 38.618125 11.5854375 0
7.9625 3.901625 0 38.618125 11.5854375 0
7.9625 3.901625 0 38.618125 11.5854375 0
7.9625 3.901625 0 38.618125 11.5854375 0
7.9625 3.901625 0 38.618125 11.5854375 0
7.9625 3.901625 0 38.618125 11.5854375 0
7.9625 3.901625 0 38.618125 11.5854375 0
7.9625 3.901625 0 38.618125 11.5854375 0
This table has 6 columns and many rows. Each row represents a single time period and the columns represent different types of discharge. So the table as a whole represents a time series of discharge quantities of different types.
The 6 columns represent:
  1. total mass of waste feed
  2. carbon mass in waste feed
  3. medicine mass in waste feed
  4. total mass of faeces
  5. carbon mass in faeces
  6. medicine mass in faeces
The mass unit can be discovered in the header properties:
inputs.FeedInputs.massUnitsSiConversionFactor
ans = '1.000000'
which tells us that the mass unit is kg (1 x SI unit). The time period represented by each row be found using:
inputs.FeedInputs.timeUnitsSiConversionFactor
ans = '1.000000'
and
inputs.FeedInputs.deltaT
ans = '3600.000000'
which indicates that the time units are seconds (1 x the SI unit for time) and that each time period comprises 3600 of these units - therefore 3600 seconds = 1 hour. So the data table represents hourly discharges in kg.
If we check the size of the table
size(inputs.data)
ans = 1×2
8760 6
we find there are 8760 entries which therefore represents 8760 hours of discharge, or 365 days. This should be consistent with the following header property:
inputs.FeedInputs.numberOfTimeSteps
ans = '8760'
Good.
