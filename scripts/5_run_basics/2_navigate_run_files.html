Navigating run files

This guide will explain how to use Matlab to navigate the various files associated with model runs.. The examples refer to organic solids runs but the points generalise to EmBZ runs.

run = project.solidsRuns.number(1)

Run inputs

Each run has an inputs file associated with it which defined the nature and properties of the the discharges that arise from the cages. This file is essentially where we define the biomass or feed rates that define the scale of the farm.
The inputs file for a given run can be located at this location
run.inputsFilePath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\inputs\bay_of_fish-1-NONE-allCages.depomodinputsproperties'
and the associated information can be initialised in Matlab using
inputs = run.inputsFile

Run configuration file

Similarly the run configuration file is found at
run.configPath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-1-NONE.depomodconfigurationproperties'
and read in as
config = run.configurationFile
config =
PropertiesFile with properties:

path: 'C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-1-NONE.depomodconfigurationproperties'
ParticleWriter: [1×1 struct]
RelseaseManager: [1×1 struct]
Model: [1×1 struct]
Transports: [1×1 struct]
This file includes configuration for optimisation runs as well as defining required model outputs such as multi-time point outputs.
Reading the file is easy. e.g.
config.Model.maxNumberOfModelIterations
ans = '52'
Note, the user interface uses a single configuration file for all runs by default. The Matlab library creates individual configuration files for every run.

Run physical properties file

Each run has a physical properties which describes the physical parameters used in the respective model simulation. This includes information on dispersion levels, settling, erosion and bed consolidation behaviour.
The file for a given run is located at

Run runtime file

Each model run has a runtime file at
run.runtimePath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-1-NONE.depomodruntimeproperties'
This file controls which of the above configuration files are to be associated with the model run. By default, Matlab based projects have individual configuration and physical properties files for every run and the runtime file is configured to point to them. In user interface projects, the runtime file for each run often points to common configuration and physical properties files which are shared between runs.

Run sur files

Sur files are output files that only existing following a successful model simulation.
Every model run has a sur file describing the distribution of organic solids at the end of the model run. Every run also has a counterpart file for describing the distribution of carbon. EmBZ runs additionally have a sur file describing the distribution of EmBZ over the domain at the same point in time.
These files can be found at the following locations
run.solidsSurPath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\results\bay_of_fish-1-NONE-N-solids-g0.depomodresultssur'
run.carbonSurPath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\results\bay_of_fish-1-NONE-N-carbon-g0.depomodresultssur'
These files can be read in to Matlab from which a variety of analyses can be made.
sur = run.solidsSur
sur =
Solids with properties:

rawDataValueCol: 'outCol1'
defaultPlotLevels: [4 192 1553 10000]
defaultUnit: 'g m^{-2}'
path: 'C:\newdepomod_projects\bay_of_fish\depomod\results\bay_of_fish-1-NONE-N-solids-g0.depomodresultssur'
rawData: [1×1 struct]
interpolatedGrid: []
X: [80×1 double]
Y: [80×1 double]
Z: [80×80 double]
GeoReferenced: 1
sur.area(250) % m2
ans = 56878.5241655428
sur.max % g m-2
ans = 23801.51202
These outputs typically are not sufficient and a greater degree of control is required. The following method allows specific sur files to be cherry picked according to their type (solids, carbon, chemical), decay basis (g=0, g=1), and time point (t)
sur = run.sur('type', 'carbon', 'g', 0)
sur =
Solids with properties:

rawDataValueCol: 'outCol1'
defaultPlotLevels: [4 192 1553 10000]
defaultUnit: 'g m^{-2}'
path: 'C:\newdepomod_projects\bay_of_fish\depomod\results\bay_of_fish-1-NONE-N-carbon-g0.depomodresultssur'
rawData: [1×1 struct]
interpolatedGrid: []
X: [80×1 double]
Y: [80×1 double]
Z: [80×80 double]
GeoReferenced: 1
A much wider range of functionality is available in this regard and is documented elsewhere.
