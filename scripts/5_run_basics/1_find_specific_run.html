Find specific run

This guide explains how to use Matlab to navigate and retrieve specific model simulations

Run collections

Each project contains 3 types of run collection
project
project =
Project with properties:

version: 2
location: [1×1 NewDepomod.PropertiesFile]
bathymetry: [1×1 NewDepomod.BathymetryFile]
flowmetry: [1×1 NewDepomod.FlowmetryFile]
name: 'bay_of_fish'
path: 'C:\newdepomod_projects\bay_of_fish'
solidsRuns: [1 Depomod.Run.Collection]
EmBZRuns: [1 Depomod.Run.Collection]
TFBZRuns: [1 Depomod.Run.Collection]
Notice that there are solidsRuns, EmBZRuns and TFBZRuns, each (in this case) wtih a single run.
We can discern this also by using the following
project.solidsRuns.size
ans = 1
When we create more runs, these will appear in these collections.

Accessing individual runs

There are a number of ways to acces individual runs. Each run is described a label and a number. We can use a few accessor functions to retrieve a particular run on this basis.
We can use the run label
run = project.solidsRuns.label('1');
Or we can use the run number
run = project.solidsRuns.number(1);
Alternatively, we can use the run index (i.e. the order they appear in the collection)
run = project.solidsRuns.item(1)

In each of these cases the same run is returned. That is because we only have one run, and it is labelled '1'.
Note that in this case the run label and the run number resemble one another, though the label is a char type and the number is a double. This doesn't have to be the case - the label can take on any alpha-numeric form, and Matlab will still assign a run number, so runs can be tracked using either basis.
The first and last runs in the collection can be accessed using the convenience methods
project.solidsRuns.first;
project.solidsRuns.last;

Run attributes

Some basic attributes can be read from the run object
run.number
ans = 1
run.label
ans = '1'
Note also that these runs are of a particular type
class(run)
ans = 'NewDepomod.Run.Solids'
