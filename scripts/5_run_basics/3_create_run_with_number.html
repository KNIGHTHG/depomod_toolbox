Creating a new run

This guide explains how to use Matlab to create a new model run

About

Runs can be labelled with any alpha-numeric label. However, one suggested approach to this is to simply label runs numerically. This is handled very easily by the Matlab library.

Create a new run

A new run can be created easily, as follows
run = project.solidsRuns.new
run =
Solids with properties:

modelFile: [1×1 NewDepomod.PropertiesFile]
physicalPropertiesFile: [1×1 NewDepomod.PropertiesFile]
configurationFile: [1×1 NewDepomod.PropertiesFile]
runtimeFile: [1×1 NewDepomod.PropertiesFile]
inputsFile: [1×1 NewDepomod.InputsPropertiesFile]
iterationInputsFile: [0×0 NewDepomod.InputsPropertiesFile]
exportedTimeSeriesFile: [0×0 NewDepomod.TimeSeriesFile]
consolidatedTimeSeriesFile: [0×0 NewDepomod.TimeSeriesFile]
solidsSur: [0×0 Depomod.Sur.Solids]
carbonSur: [0×0 Depomod.Sur.Solids]
label: '2'
iterationRunNumber: []
modelFileName: 'bay_of_fish-2-NONE.depomodmodelproperties'
tide: 'N'
resultsPath: 'C:\newdepomod_projects\bay_of_fish\depomod\results'
project: [1×1 NewDepomod.Project]
cfgFileName: []
number: 2
cages: [2 Depomod.Layout.Site]
There are a few things to notice here. The new run has a run number of 2 and a run label of '2'. This is because it is now the second run in the solidsRuns collection and has been labelled accordingly. Subsequent runs would take subsequent, sequential integer labels.
The new run has a full complement of configurations files. These files are currently identical to the previous run (run '1') albeit with the required changes to reflect the fact that this is a distinct run. These configuration files can be changed if something different is required in the new run.
In general, creating a new run in this manner will cause the new run to have a number/label that is a simple increment over the previous last run. But the previous last run represents the template for the new run in terms of configuration.
We can check that the new run is referencing new files
run.cagesPath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\cages\bay_of_fish-2.depomodcagesxml'
run.inputsFilePath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\inputs\bay_of_fish-2-NONE-allCages.depomodinputsproperties'
run.physicalPropertiesPath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-2-NONE.depomodphysicalproperties'
Notice that these paths now reference run 2 (e.g. \bay_of_fish-2-)

Using a specific template run

If the new run should have a configuration (e.g. run times, physical properties, etc.) that is based on a different run from the last run in the collection, this can be explicitly specified in this way.
run = project.solidsRuns.new('template', 6)
In this case, the new run would be created and would base all configuration options on run number 6 (assuming that run exists).
