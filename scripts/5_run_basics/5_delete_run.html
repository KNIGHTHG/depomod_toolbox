Deleting a run

This guide explains how to use Matlab to delete a model run

Existing runs

In order to demonstrate the deletion of a run, let's first check how runs we have in our project
project.solidsRuns.size
ans = 3
And let's just check the labels/numbers of these runs
project.solidsRuns.labels
ans = 3×1 cell array
{'1' }
{'2' }
{'hindcast_2017'}
project.solidsRuns.numbers
ans = 3×1
1
2
3
Okay, so we have 3 runs, numbered 1,2 and 3, and the last one of which has a more descriptive alpha-numeric label.

Deleting a run

We delete the last run. Assuming we have it instantiated in Matlab
run = project.solidsRuns.number(3)

we can delete easily
run.deleteRun
Now we can check the size of the run collection again
project.solidsRuns.size
ans = 2
project.solidsRuns.labels
ans = 2×1 cell array
{'1'}
{'2'}
project.solidsRuns.numbers
ans = 2×1
1
2
So the run we had selected has been deleted and no longer appears in the run collection

Note

Deleting runs in this manner deletes associated run files, with a couple of exceptions. The cage file is not deleted by default because this may be shared with a similarly numbered or labelled run in another collection (e.g. EmBZ runs). This can be forced by using
run.deleteRun('cages',1)
Output files are also not deleted using this method. This is simply for the reason that these may be very numerous and take a wide range of different file names. This is intended to be supported in the future. This does not cause any issues with navigating the remaining runs and only causes a problem if a new run is created using the same label as a deleted run - from the perspective of Matlab (and the NewDepomod software), the older results files look like they're associated with the new run. Care is therefore required.
