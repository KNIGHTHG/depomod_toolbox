Introduction to physical properties

This guide describes some pertinent physical properties that are useful in configuring model runs

The physical properties file

The physical properties file associated with a particular run (when using the Matlab library convention) is located at the following location
run.physicalPropertiesPath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-1-NONE.depomodphysicalproperties'
The file can be instantiated in Matlab using
physicalProperties = run.physicalPropertiesFile

Initial settling

Characteristics of particles arising from the farm cages can be configured. Settling velocities for these particles (waste feed and faeces) are configured in the inputs properties file. Other properties can be configured in the physical properties file.
The model uses a random walk implementation to represent the effects of random dispersive processes such as turbulence. The following properties describe the specific properties used to define the magnitude of this random dispersion and which refer to the x, y (both horizontal), and z (vertical) dimensions respectively.
physicalProperties.Transports.suspension.walker.dispersionCoefficientX
ans = '0.1'
physicalProperties.Transports.suspension.walker.dispersionCoefficientY
ans = '0.1'
physicalProperties.Transports.suspension.walker.dispersionCoefficientZ
ans = '0.001'
The x and y values should be identical. The type of random walk implemention is defined using
physicalProperties.Transports.suspension.walker.type
ans = 'LATTICEWALKER'
which can take values of LATTICEWALKER or UNIFORMWALKER.
Particles can be configured to have their settling velicities modified by velocity shear in the water column
physicalProperties.Transports.suspension.settling.modifiedSettling
ans = 'false'
physicalProperties.Transports.suspension.settling.allowBuoyant
ans = 'false'
These properties can be set to true or false. It is recommended that they are set to false.

Erosion

Erosion occurs when the shear stress at the bed exceeds a particular threshold. This threshold is defined at:
physicalProperties.Transports.BedModel.tauECritMin
ans = '0.02'
Shear stress is calculated according to the lowermost flow record and one of two calculation methods:
physicalProperties.Transports.regime.frictionvelocity.type
ans = 'CLAUSERCHART'
This property can be specified as either LAWOFTHEWALL or CLAUSERCHART. These calculation methods refer to a concept know as the bed "roughness length" or "roughness height" or sometimes known as z0. This parameter is defined in the physical properties file at:
physicalProperties.Transports.bottomRoughnessLength.smooth
ans = '0.00003'
When erosion takes place, the quantity eroded is limited by a function defined by the following parameters:
physicalProperties.Transports.BedModel.massErosionCoefficient
ans = '0.031'
physicalProperties.Transports.BedModel.massErosionExponent
ans = '1'

Resuspension

Eroded particles become resuspended within the overlying water column. Particles are created from the aggregated seabed mass within each model domain cell. The number of particles created to rerpesent this process is determined by this parameter
physicalProperties.Transports.BedModel.releaseParticles.particlesPerArea
ans = '0.0064'
and particles are created with a height above the seabed defined by
physicalProperties.Transports.BedModel.releaseHeight.height
ans = '0.12'
The settling velocity of the resuspended sediment is defined by the following distribution parameters
physicalProperties.Transports.BedModel.settlingVelocityOfSediment.location
ans = '-0.0057'
physicalProperties.Transports.BedModel.settlingVelocityOfSediment.dispersion
ans = '0.00057'
physicalProperties.Transports.BedModel.settlingVelocityOfSediment.distribution
ans = 'UNIFORM'
Not that the settling velocity central tendency (.location) is defined negatively.
These settling veloocities can be configured to be modified according to velocity shear in the water column.
physicalProperties.Transports.resuspension.settling.modifiedSettling
ans = 'false'
physicalProperties.Transports.resuspension.settling.allowBuoyant
ans = 'false'
It is recommended that these are set to false.
As with particles settling from the cages, resuspended particles undergo random walk according to analogous (but separately defined) parameters.
physicalProperties.Transports.resuspension.walker.dispersionCoefficientX
ans = '0.1'
physicalProperties.Transports.resuspension.walker.dispersionCoefficientY
ans = '0.1'
physicalProperties.Transports.resuspension.walker.dispersionCoefficientZ
ans = '0.001'
physicalProperties.Transports.resuspension.walker.type
ans = 'LATTICEWALKER'

Bed consolidation

The consolidation of bed material can be represented in the model. This has the effect of increasing the criticial shear stress for erosion in buried (i.e. compacted) layers, which may retain (for a time) this "hardened" condition following exhumation. This requires specifying a system of bed layers according to:
physicalProperties.Transports.BedModel.numberOfLayers
ans = '10'
physicalProperties.Transports.BedModel.dLayerMass
ans = '3375'
Note that the dLayerMass property represents the mass in kg of a bed layer across a whole model domain cell.
The rate at which buried layers are compacted (following burial) and expanded (following exhumation) are determined by the following parameters
physicalProperties.Transports.BedModel.expansionT50
ans = '14400'
physicalProperties.Transports.BedModel.contractionT50
ans = '900.0'
physicalProperties.Transports.BedModel.internalFrictionAngle
ans = '23'
The internalFrictionAngle parameter defines the relationship between buried depth and critical shear stress for erosion. The contractionT50 and expansionT50 values defined the rates of contraction and expansion respectively as half-lives in seconds.

Editing and saving

As with all properties file representations in Matlab, individual properties can be edited easily:
physicalProperties.Transports.BedModel.tauECritMin = num2str(0.025)
and saved back to file
physicalProperties.toFile
Note that all specified values must be declared as char variables - even if numeric.
