Setting the run duration

This guide explains how to set the simulation duration using Matlab

Understanding simulation duration

Defining the duration of a model run requires two specific components:
  1. Specifying the appropriate discharge time series length in the run inputs properties file
  2. Setting the run start and end times in the run model properties file
For a given inputs file discharge time series, the start and end times declared in the model propreties file specify the points between which the simulation will run.
Recall, that we can identify the run duration defined in the inputs as follows:
inputs.FeedInputs.numberOfTimeSteps
ans = '8760'
size(inputs.data,1)
ans = 8760
These are hours, and therefore represent 365 days, 1 year.
If we check the model properties file, we can see when during this period (and associated discharge characteristics) the model is configured to run.
model = run.modelFile;
model.ModelTime.startTime
ans = '0'
model.ModelTime.endTime
ans = '31536000000'
So the start time is 0 and end time is 31536000000. Now, these values are in milliseconds and so they translate to 0 days and 365 days. That is, the model is configured to run from the start to the end of the discharges defined in the inputs file. The configured model duration is 365 days.

Example: a 24 hour run

These two files - presently both configured to represent 365 days - do not have to match. For example, we can configure the model to run for just 1 day by doing the following.
model.ModelTime.endTime = num2str(24*3600*100);
which sets the model end time to be 24 hours (specified as milliseconds). The start time is still 0.
Equally, we could set the start and end times as:
model.ModelTime.startTime = num2str(364*24*3600*1000);
model.ModelTime.endTime = num2str(365*24*3600*1000);
In the first case we are using the first 24 hours of the specified 365-day discharge time series and in the latter case we are using the last 24 hours. In both cases, the model simulation will last 24 hours.
This approach can be useful if a specific discharge scenario has been curated in the inputs properties file and the modeller wants to run different periods of that scenario.
Note that when making these edits, the model file must be saved.
model.toFile

Synchronising the run duration and inputs

In general it is more typical to set up the inputs discharge data and the model time configuration to be consistent with one another. For example, if we define a 118 day discharge scenario in our inputs file, then we want to set the the model file to produce a 118 day run. This can be done in one easy step by using the following method:
run.syncRunDurationWithInputs
Notice that this method is invoked on the run object. It is able to query the associated inputs file and set the appopriate run duration in the associated model properties file. There is no need to explicitly save any files when using this method.

Another helper method

An alternative method for quickly setting the run duration is to simply specify the number of days required in the run.
run.setRunDurationDays(90)
In this example the run is configuration to last for 90 days by setting the endTime property of the model properties file to 90 days (or the equivalent milliseconds). The startTime property is set to 0. This therefore assumes that the inputs discharge time series exceeds the specified duration.
