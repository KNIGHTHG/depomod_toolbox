Introduction to run configuration

This guide will give a brief introduction to the files used to configure individual runs

Run configuration files

Domain/bathymetry and flow are defined on a project wide basis. Individual runs include separate definitions for many configuration options. The individual configuration files related to individual runs are as follows:
  • cages definition
  • inputs (discharges) definition
  • model properties
  • configuration properties
  • physical properties
  • runtime properties
These files are located within a project according to the following conventions
run.cagesPath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\cages\bay_of_fish-1.depomodcagesxml'
run.inputsFilePath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\inputs\bay_of_fish-1-NONE-allCages.depomodinputsproperties'
run.modelPath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-1-NONE.depomodmodelproperties'
run.configPath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-1-NONE.depomodconfigurationproperties'
run.physicalPropertiesPath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-1-NONE.depomodphysicalproperties'
run.runtimePath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-1-NONE.depomodruntimeproperties'
Cages and inputs are dealt with in other sections of this guide. This guide is limited to describing the subsequent 4 file types: model, configuration, physical and runtime.
Note that projects created using this Matlab library have runs which are configured to use their own unique versions of each of these files. By default, the NewDepomod interface uses shared configuration properties and physical properties files across runs. This distinction should be understood. It does not effect how runs are executed in the software - only how the configuration is specified across files.

A general note about the file format and handling

The model, configuration, physical and runtime properties files all utilise the same file format - the Java properties format. These are similar to the run inputs and project flow files but without the CSV-style data table at the end of the file. That is, they are simply lists of property names and values.
All of the files can be read the same way, edited the same way and saved the same way.

Any of these files can be read in by passing their full file path to the NewDepomod.PropertiesFile constructor method.
physicalProperties = NewDepomod.PropertiesFile('C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-1-NONE.depomodphysicalproperties')

Editing a property

physicalProperties.Transports.BedModel.massErosionCoefficient = num2str(0.04);
Note that all values must be char types, even if numeric.
Checking the change we made:
physicalProperties.Transports.BedModel.massErosionCoefficient
ans = '0.04'

Saving changes

physicalProperties.toFile
will save the changes back to the original file. To save to a separate location:
physicalProperties.toFile('path\to\file.depomodphysicalproperties')
