Setting multiple outputs

This guide explains how to configure the model to output seabed informatin at numerous time points

Background context

By default, the NewDepomod software outputs seabed impact information to file only at the end of the simulation. The outputs are of the format .depomodresultssur and take the following types of filename.
run.solidsSurPath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\results\bay_of_fish-1-NONE-N-solids-g0.depomodresultssur'
run.carbonSurPath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\results\bay_of_fish-1-NONE-N-carbon-g0.depomodresultssur'
These are typically not sufficient to draw conclusions regarding seabed impacts and it is therefore necessary to output more "sur" files from multiple time points during the model run.

Specifying additional outputs

Additional outputs are specified within the run configuration properties file. This file can be found easily using:
run.configPath
ans = 'C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-1-NONE.depomodconfigurationproperties'
Reading this file into Matlab for manipulation is using using:
config = run.configurationFile
config =
PropertiesFile with properties:

path: 'C:\newdepomod_projects\bay_of_fish\depomod\models\bay_of_fish-1-NONE.depomodconfigurationproperties'
RelseaseManager: [1×1 struct]
Model: [1×1 struct]
ParticleWriter: [1×1 struct]
Transports: [1×1 struct]
Multiple outputs through the model run duration are specified using the Transports.recordTimes property. This property requires a values describing the required points in time, specified in seconds, separated by commas.

A manual example

For example, suppose, within out 365 day model run, that we want seabed outputs to be written to file on the 100th, 200th and 300th day, we would need to do something like the following.
Specify the times required
requiredDays = [100, 200, 300]
requiredDays = 1×3
100 200 300
Convert to seconds
requiredSeconds = requiredDays.*(24*3600)
requiredSeconds = 1×3
8640000 17280000 25920000
Convert to char variables and join, delimited using commas.
recordTimesString = strjoin(arrayfun(@num2str, requiredSeconds, 'un',0),',')
recordTimesString = '8640000,17280000,25920000'
Now, set in the appropriate property in the configuration properties file
config.Transports.recordTimes = recordTimesString;
Now, let's check.
config.Transports.recordTimes
ans = '8640000,17280000,25920000'
Okay. The next step is the write back to the file so that the information can be used when the model is executed.
config.toFile
ans = '0'
ans = '31536000000'

Some helper methods

Configuring many output times can be complicated and time consuming so there are available a number of convenience methods that help to set the recordTimes property value more easily. There are several permutations.
To clear the record times we can use
run.clearOutputTimes;
Let's take a look
config.Transports.recordTimes
ans =
0×0 empty char array
Empty, as expected.
The most general helper method is .setOutputTimes() executed from the model run object. We can use this to set an output window and resolution. For example, to set 3-hourly outputs we can use:
run.setOutputTimes('resolution', 3*3600);
Note that the resolution is specified in seconds.

A couple more methods are a available that make some tasks easier.
run.clearOutputTimes;
run.setDailyOutputTimes;
sets one output per day for the entirety of the model run.
run.clearOutputTimes;
run.setOutputTimesFromEnd(7, 3*3600);
sets outputs over a specified period at the end of the run. The period is defined in days, as the first argument of the function. So in the case above, outputs are set over the final 7 days. The second argument is the resolution in seconds - in this case, 3-hourly.
