Read bathymetry properties file

This guide explains how to use Matlab to read in an existing .depomodbathymetryproperties file.

About bathymetry properties files

The NewDepomod bathymetry file defines the model domain. The file is located with the \bathymetry folder of the project, is named after the project name and has a file extension .depomodbathymetryproperties.
The file follows the Java Properties format and includes a CSV-style data table at the end containing the cell-by-cell bathymetry data. Specifically, the information in the files includes:
- the domain bounds (OSGB easting/northing)
- the grid resolution (i.e. number of grid cells in x and y dimensions)
- data grid of water depth values for each cell (i.e. the bathymetry)
The data grid is negative downwards and values of 10 denote land. The NewDepomod.BathymetryFile Matlab object can be used to read, view, edit and save the bathymetry information and data.
Note that this guide only describes the regular grid based bathymetry description which uses the .depomodbathymetryproperties format. Irregular, triangular grids can also be used in NewDepomod.

Initialise a bathymetry file object within a project

If you have a whole project set up then the bathymetry file can be instantiated in Matlab by taking advantage of the navigation provided by the NewDepomod.Project class, without needing file paths etc.
projectDir = 'C:\newdepomod_projects\bay_of_fish';
project = NewDepomod.Project.create(projectDir);
bathy = project.bathymetry
bathy =
BathymetryFile with properties:

GridgenBathymetryFile: [0×0 AutoDepomod.BathymetryFile]
GridgenDomainFile: [0×0 AutoDepomod.DomainFile]
data: [79×79 double]
path: 'C:\newdepomod_projects\bay_of_fish\depomod\bathymetry\bay_of_fish.depomodbathymetryproperties'
Domain: [1×1 struct]
endOfDataMarker: 'endOfDataMarker'
startOfDataMarker: 'startOfDataMarker'
This returns an object which holds all of the specified bathymetry information.

Initialise a bathymetry file directly from file path

Alternatively, a bathymetry file can be instantiated directly by using the direct file path, e.g.
bathy = NewDepomod.BathymetryFile('C:\newdepomod_projects\bay_of_fish\depomod\bathymetry\bay_of_fish.depomodbathymetryproperties')
bathy =
BathymetryFile with properties:

GridgenBathymetryFile: [0×0 AutoDepomod.BathymetryFile]
GridgenDomainFile: [0×0 AutoDepomod.DomainFile]
data: [79×79 double]
path: 'C:\newdepomod_projects\bay_of_fish\depomod\bathymetry\bay_of_fish.depomodbathymetryproperties'
Domain: [1×1 struct]
endOfDataMarker: 'endOfDataMarker'
startOfDataMarker: 'startOfDataMarker'
Now we've got the same object but by simply passing in its individual file path. This can be done for any .depomodbathymetryproperties file - just pass in the path as above to get a Matlab representation of the file which can be read, edited and saved easily.

Read the bathymetry information

We can read the information from the bathymetry object.
bathy.Domain.spatial
ans = struct with fields:
reference: [1×1 struct]
minX: '349340'
maxX: '351340'
minY: '1067960'
maxY: '1069960'
bathy.Domain.data
ans = struct with fields:
elementFormat: 'csvRegularGrid'
numberOfElementsX: '79'
numberOfElementsY: '79'
These give us the domain bounds (easting/nothing) and the resolution of the domain. Individual values can be drilled into.
bathy.Domain.spatial.maxX
ans = '351340'
